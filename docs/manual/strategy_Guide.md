## ２. 設定の優先順位と `-I` オプション

Tickwise v1.1 では、検証の効率化のために以下の優先順位を徹底しています。

1. **コマンドオプション**（最優先）
2. **tickwise.env**（ファイル設定）
3. **デフォルト値**（コード内）

### 💡 レシピ検証の極意：`-I` オプション
`.env` の中身をいちいち書き換える必要はありません。**`-I` (Ignore) を付与すれば、すべての指標設定がリセットされ、コマンドラインで指定した「レシピ」のみが純粋に実行されます。** 以下のレシピはすべて `-I` 併用を前提としています。

## 3. 実戦汎用レシピ 10選

### ① 【順張り】王道のトレンドフォロー
上昇トレンドの波に乗り、勢いが衰えていないかを確認する最も標準的な設定です。
```bash
tickwise -t <TICKER> -I --stance buyer --ema --sma --adx --buy-rsi 40 --weight-ema 2.0 --weight-sma 2.0 --weight-adx 2.5
```
- **論理**: 複数の移動平均のクロスを主軸にし、ADXでトレンドの「本気度」を判定。RSIは押し目を拾うために少し高めの40に設定。

- **結果の見方 (返値の指南)**
    - **EMA/SMAスコア**: `+1`〜`+2` が出ているのが理想。メッセージに「ゴールデンクロス進行中」とあれば順調。`0` やマイナスならトレンド失速。
    - **ADXスコア**: `+1` (25以上) または `+2` (40以上) が出ているかを確認。数値が上昇中なら「トレンドの継続性」が高い。
    - **総合判定（ゲージ）**: `🟢 強い買い` または `🟢 買い優勢` を目指す。ADXが低くEMAだけが良い場合は「一時的なリバウンド」の可能性を考慮する。

- **判断**：移動平均の形（スコア）が良く、ADXが `+1` 以上で、RSIが40以上を維持 → このレシピの基準クリア。

### ② 【順張り】一目均衡表・雲の上の戦い
中長期のトレンドが「厚いサポート」に支えられているかを重視します。
```bash
tickwise -t <TICKER> -I --stance holder --ichimoku --sma --weight-ichimoku 3.0 --weight-sma 1.0
```
- **論理**: 一目均衡表の転換線・基準線の関係を最重視（Weight 3.0）。雲の上にいる間の「安心感」を数値化します。

- **結果の見方 (返値の指南)**
    - **Ichimokuスコア**: `+2` (三役好転に近い状態) なら非常に強気。`0` 以下は雲の中、または雲下への脱落を示唆。
    - **分析メッセージ**: 「価格が雲の上で推移し、強い支持帯として機能」という記述があるか注目。基準線を割り込むとスコアが削られます。
    - **総合判定 (holder)**: ホルダー視点では、ゲージが `🟡 買い気配` 以上の維持を確認。`⚪️ 様子見` 以下に落ちるなら、支持帯の崩壊を警戒する。

- **判断**: 雲上維持 ＋ 転換線＞基準線 ＋ SMA `+1` 以上 → 保有継続の根拠が強い。

### ③ 【順張り】ボラティリティ・ブレイクアウト
ボリンジャーバンドが収束（スクイーズ）し、パワーを貯めた後の爆発を狙います。
```bash
tickwise -t <TICKER> -I --bollinger --bb-bandwidth-squeeze-pct 8 --weight-bollinger 3.0 --adx --weight-adx 2.0
```
- **論理**: バンド幅が8%以下の「嵐の前の静けさ」を検知。放たれた瞬間のADXの立ち上がりを評価します。

- **結果の見方 (返値の指南)**
    - **Bollingerメッセージ**: `⚠️ Bandwidthが8.00%以下に収束（スクイーズ）→ 抜けた方向への強いトレンド発生に注意` という警告が出ているか。
    - **Bollingerスコア**: 上側バンドを突破し始めると `+1`〜`+2` に跳ねる。スクイーズ中（横ばい）は `0` になりやすい。
    - **ADXスコア**: ブレイクと同時に ADX が上昇し `+1` (25越え) になれば、そのブレイクは「本物」である確率が高い。

- **判断**: スクイーズ警告あり ＋ 価格が上バンド接近 ＋ ADX上昇開始 → ブレイクアウト狙いのエントリー好機。

### ④ 【順張り】短期決戦・モメンタム重視
価格の変化率（ROC）を主役に据え、短期間で一気に駆け上がる銘柄を追います。
```bash
tickwise -t <TICKER> -I --roc --weight-roc 3.0 --ema --weight-ema 1.0 --news-freshness pd
```
- **論理**: 10日間のROCが急騰していることを重視。ニュースは `pd`（過去24時間）に絞り、今の熱量を測ります。

- **結果の見方 (返値の指南)**
    - **ROCスコア**: 急騰時は `+2` に張り付く。スコアが `+2` から `+1` に落ちた時は、勢いのピークアウト（短期的な利確時）を検討。
    - **ニュースハイライト (LLM)**: 直近24時間 (`pd`) のニュースが「ポジティブなサプライズ」であるかを確認。
    - **総合判定**: モメンタム戦略なので `🟢 強い買い` が出ている間だけ乗る。スコアが急速に減衰し始めたら深追いしない。

- **判断**: ROCスコア `+2` ＋ EMA `+1` 以上 ＋ 直近ニュースが追い風 → 短期追随。

### ⑤ 【逆張り】売られ過ぎ反発（基本形）
教科書的な「売られ過ぎ」からのリバウンドを論理的に拾います。
```bash
tickwise -t <TICKER> -I --stance buyer --buy-rsi 30 --macd-minus-ok --macd-diff-low 1.0 --weight-basic 2.5
```
- **論理**: RSI 30以下を狙いつつ、`macd-minus-ok` でマイナス圏での反転を許容。`diff-low` を絞ることで下げ止まりを確認します。

- **結果の見方 (返値の指南)**
    - **RSI値**: `30` 以下の数値が出ているか。`20` を切ると「極端な売られ過ぎ」として反発期待が高まるが、ズルズル下げるリスクも併存。
    - **MACDメッセージ**: `--macd-diff-low 1.0` により、MACDとシグナルの差が1.0未満なら「収束（下げ止まり）」と判定される。ここが `0` 付近ならリバウンドの準備完了。
    - **総合判定**: `🟡 買い気配` など、マイナスからプラスに転じ始めた瞬間（初動）が出ているかをゲージで見る。

- **判断**: RSI ≤ 30 ＋ MACD差が1.0以下に収束 → 「リバウンドの土台」が成立。

### ⑥ 【逆張り】パニック売り・底値拾い
市場が総悲観になり、セリングクライマックスに達した瞬間を狙う過激な設定です。
```bash
tickwise -t <TICKER> -I --stance buyer --buy-rsi 20 --bollinger --weight-bollinger 2.5 --macd-minus-ok --weight-basic 1.5
```
- **論理**: RSI 20という極限状態を設定。ボリンジャーバンドの下限（-2σ）を突き抜けた後の戻りを目指します。

- **結果の見方 (返値の指南)**
    - **RSI値**: `20` 以下のパニック水準を確認。
    - **Bollingerスコア**: `-2` (下側バンドを下抜け) が出ている間はまだ「落下の最中」。これが `-1` や `0` に戻ったタイミング（バンド内回帰）が絶好の底拾いポイント。
    - **総合判定**: ゲージがまだ `🔴` や `🟠` でも、LLM総評で「下げ渋り」「乖離の修正」に言及があれば、打診買いの検討材料。

- **判断**: RSI ≤ 20 ＋ 下バンドからの回帰メッセージ ＋ MACD悪化停止 → 逆張りエントリー検討。

### ⑦ 【逆張り】VWAP回帰（平均回帰戦略）
適正価格（VWAP）から大きく乖離した株価が、磁石のように引き戻される性質を利用します。
```bash
tickwise -t <TICKER> -I --vwap --weight-vwap 3.0 --buy-rsi 35 --stance buyer
```
- **論理**: VWAPと現在値の乖離をスコアの最大根拠にします。行き過ぎたゴムが戻る瞬間を捉えます。

- **結果の見方 (返値の指南)**
    - **VWAPスコア**: `+2` が出ているなら、統計的に「売られすぎ（下方に大きく乖離）」を意味する。
    - **価格とVWAP**: 出力される `VWAP: XXX.X` に対して現在値がどれほど低いかを見る。乖離率が縮小し始めたら回帰のサイン。
    - **総合判定**: ゲージが `🟡` 以上に浮上してきたら、平均価格への収束（リバウンド）を狙える局面。

- **判断**: VWAPスコア `+2` ＋ RSI ≤ 35 ＋ 乖離縮小の兆し → 回帰狙い。

### ⑧ 【防御】ホルダーの警戒モード
保有株の「逃げどき」を探る、ディフェンス重視の設定です。
```bash
tickwise -t <TICKER> -I --stance holder --sell-rsi 65 --macd-diff-mid 5 --bollinger --weight-bollinger 2.0
```
- **論理**: RSIの売り閾値を少し下げて早めに警告。バンド上限での過熱感を厳しめに評価させます。

- **結果の見方 (返値の指南)**
    - **RSI値**: `65` 以上（設定値越え）でメッセージが赤く強調される。これは「利確準備」のシグナル。
    - **Bollingerメッセージ**: `価格が上側バンドを突破（過熱圏）` という表示が出たら注意。特にその後バンド内へ押し戻されたら天井の可能性。
    - **総合判定 (holder)**: ゲージが `🟠 売り気配` 方向へ振れ始めたら、利益が残っているうちの一部売却などを検討するタイミング。

- **判断**: RSI ≥ 65 ＋ 上バンド反落 ＋ MACD差の縮小 → 利益確定の検討。

### ⑨ 【全方位】AIセカンドオピニオン
全指標をフル稼働させ、AIに「指標間の不整合」を炙り出させます。
```bash
tickwise -t <TICKER> -I --ema --sma --bollinger --fibonacci --adx --ichimoku --vwap --llm
```
- **論理**: 人間では見きれない多角的データをAIにぶつけ、総合的な「穴」を指摘させるためのレーダーチャート設定。

- **結果の見方 (返値の指南)**
    - **スコアの不整合をチェック**: 例えば「トレンド系 (+2)」なのに「オシレーター系 (-2)」などの矛盾を探す。これが「勢いはあるが過熱している」というAIの解説の根拠になる。
    - **Fibonacci/一目均衡表**: 分厚い雲や目標価格（38.2%戻しなど）が重なっているポイント（重合）をAIが指摘しているか確認。
    - **LLM総評**: 結論の「買い/売り」よりも、そのプロセスでAIが「懸念点」として挙げている指標を特定し、自分の判断の死角を埋める。

- **判断**: 全指標のスコアが同方向 ＝ 自信度高。スコアが割れている ＝ 短期と中期のシナリオを分けて考える必要あり。

### ⑩ 【情報戦】材料・テーマ深掘り
テクニカルはベースに留め、ニュースの「中身」をAIに徹底的に読み込ませます。
```bash
tickwise -t <TICKER> -I --llm --news -q "業績予想 修正 増配" --weight-news 3.0 --news-count 20
```
- **論理**: カスタムクエリ `-q` で材料を絞り込み、AIの総評を「情報先行」の投資判断に活用します。

- **結果の見方 (返値の指南)**
    - **ニュースハイライト (LLM)**: AIがニュースを「Tier A（一次情報）」〜「Tier C（参考）」に仕分けしている。Tier Aが多く、かつ「価格影響：高」と判定されているかを見る。
    - **Weight-News**: ニュースの重みを3.0にしているため、テクニカルが弱くても材料一本で総合スコアが「買い」に振ることがある。この場合「材料株」としてのリスクを承知の上で乗るか判断。
    - **総評**: AIが「織り込み済み」と判断しているか、それとも「さらなる上値」を予想しているかのニュアンスを読み取る。

- **判断**: Tier Aニュースの集積 ＋ AIが「さらなる価格反映」を示唆 → 情報先行のエントリー。
