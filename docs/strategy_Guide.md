## ２. 設定の優先順位と `-I` オプション

Tickwise v1.1 では、検証の効率化のために以下の優先順位を徹底しています。

1. **コマンドオプション**（最優先）
2. **tickwise.env**（ファイル設定）
3. **デフォルト値**（コード内）

### 💡 レシピ検証の極意：`-I` オプション
`.env` の中身をいちいち書き換える必要はありません。**`-I` (Ignore) を付与すれば、すべての指標設定がリセットされ、コマンドラインで指定した「レシピ」のみが純粋に実行されます。** 以下のレシピはすべて `-I` 併用を前提としています。

## 3. 実戦汎用レシピ 10選

### ① 【順張り】王道のトレンドフォロー
上昇トレンドの波に乗り、勢いが衰えていないかを確認する最も標準的な設定です。
```bash
tickwise -t <TICKER> -I --stance buyer --ema --sma --adx --buy-rsi 40 --weight-ema 2.0 --weight-sma 2.0 --weight-adx 2.5
```
- **論理**: 複数の移動平均のクロスを主軸にし、ADXでトレンドの「本気度」を判定。RSIは押し目を拾うために少し高めの40に設定。

- **結果の見方**
EMA / SMA：短期＞中期＞長期の並び（上向きの“順パーフェクトオーダー”）や、直近での上抜け（ゴールデンクロス寄り）が強い。逆に短期が下に潜る・横ばい化は失速サイン。

ADX：上昇（右肩上がり）が最重要。水準は一般に「25以上＝トレンドが効きやすい」目安。低位で横ばいなら“トレンド未成立”として過信しない。

RSI（買い閾値40）：40付近で反発・維持できているなら“押し目からの再加速”として扱いやすい。40を明確に割り込んで戻らないなら、レシピの前提（順張り）が崩れやすい。

- **判断**：移動平均の形が良く、ADXが上向きで、RSIが40以上を維持 → このレシピの「順張り条件が揃っている」。どれかが崩れるなら“見送り or サイズ縮小”寄り。

### ② 【順張り】一目均衡表・雲の上の戦い
中長期のトレンドが「厚いサポート」に支えられているかを重視します。
```bash
tickwise -t <TICKER> -I --stance holder --ichimoku --sma --weight-ichimoku 3.0 --weight-sma 1.0
```
- **論理**: 一目均衡表の転換線・基準線の関係を最重視（Weight 3.0）。雲の上にいる間の「安心感」を数値化します。

- **結果の見方**
雲（先行スパン）：価格が雲の上にあり、雲が上向き（将来の支持帯が厚い）なら強い。雲の中は“方向感なし”、雲下は“警戒”。

転換線/基準線：転換線＞基準線（上向き）が継続しているか。クロスして下回ると、ホルダー目線では“守りに寄せる”合図。

SMA：一目が良くてもSMAが下向きなら“中期がまだ弱い”可能性。SMAも上向きなら安心度が上がる。

判断（holder）：雲上＋転換線＞基準線＋SMA上向き → “保有継続の根拠が強い”。雲割れ/基準線割れが重なるなら“警戒モード”へ。

### ③ 【順張り】ボラティリティ・ブレイクアウト
ボリンジャーバンドが収束（スクイーズ）し、パワーを貯めた後の爆発を狙います。
```bash
tickwise -t <TICKER> -I --bollinger --bb-bandwidth-squeeze-pct 8 --weight-bollinger 3.0 --adx --weight-adx 2.0
```
- **論理**: バンド幅が8%以下の「嵐の前の静けさ」を検知。放たれた瞬間のADXの立ち上がりを評価します。

- **結果の見方**
BandWidth（スクイーズ）：8%以下が“溜め”。重要なのは その後にBandWidthが拡大へ転じること（拡大しないならブレイク未成立）。

価格とバンド：上抜け狙いなら上側バンドを終値で突破→バンド外で推移、が素直。すぐバンド内へ戻ると“ダマシ”になりやすい。

ADX：スクイーズ解除と同時にADXが上向くとブレイクに“持続力”が出やすい。ADXが低位のままだと一過性になりやすい。

- **判断**：スクイーズ（8%以下）→ BandWidth拡大 → 価格がバンド方向に張り付く → ADX上向き、の順に揃うほど信頼度が上がる。

### ④ 【順張り】短期決戦・モメンタム重視
価格の変化率（ROC）を主役に据え、短期間で一気に駆け上がる銘柄を追います。
```bash
tickwise -t <TICKER> -I --roc --weight-roc 3.0 --ema --weight-ema 1.0 --news-freshness pd
```
- **論理**: 10日間のROCが急騰していることを重視。ニュースは `pd`（過去24時間）に絞り、今の熱量を測ります。

- **結果の見方**
ROC：プラスで上昇（加速）しているほど強い。ROCが高水準でも“頭打ち（伸びが鈍化）”が出ると短期では天井圏の可能性が上がる。

EMA：モメンタムが強くても、価格がEMAを割り込むと“短期の勢いが折れた”サインになりやすい。EMA上で推移できているかをチェック。

ニュース（pd）：24時間内の材料が“追い風”か、“悪材料”かをまず分類。モメンタム型はニュースと逆行し始めると崩れが速い。

- **判断**：ROCの加速＋EMA上維持＋直近ニュースが追い風 → “短期追随の形”。ROC鈍化やEMA割れが出たら“利確/撤退”判断が優先されやすい。

### ⑤ 【逆張り】売られ過ぎ反発（基本形）
教科書的な「売られ過ぎ」からのリバウンドを論理的に拾います。
```bash
tickwise -t <TICKER> -I --stance buyer --buy-rsi 30 --macd-minus-ok --macd-diff-low 1.0 --weight-basic 2.5
```
- **論理**: RSI 30以下を狙いつつ、`macd-minus-ok` でマイナス圏での反転を許容。`diff-low` を絞ることで下げ止まりを確認します。

- **結果の見方**
<T> RSI（30）：30以下は“売られ過ぎ領域”。ただし“下落トレンド中の30割れ”は継続し得るので、次のMACD条件で補強する。

MACD（minus-ok）：MACDがマイナス圏でも良い＝“底値圏の反転初動”を拾う設計。重要なのは MACDが悪化し続けていないこと（下げ加速中は危険）。

MACD diff（--macd-diff-low 1.0）：Tickwiseが出す「diff」が 設定値以下 に収束しているなら、“勢いの弱まり（下げ止まり寄り）”として扱う。diffが広がるなら“まだ荒れている”。

- **判断**：RSI≤30 かつ diff≤1.0（収束） → “反発の土台”。RSIだけでdiffが大きいなら“落下中のナイフ”寄り。

### ⑥ 【逆張り】パニック売り・底値拾い
市場が総悲観になり、セリングクライマックスに達した瞬間を狙う過激な設定です。
```bash
tickwise -t <TICKER> -I --stance buyer --buy-rsi 20 --bollinger --weight-bollinger 2.5 --macd-minus-ok --weight-basic 1.5
```
- **論理**: RSI 20という極限状態を設定。ボリンジャーバンドの下限（-2σ）を突き抜けた後の戻りを目指します。

- **結果の見方**
RSI（20）：極端な売られ過ぎ。ここは“反発候補”であって“安全”ではない（継続下落の余地もある）。

ボリンジャー：下側バンドを突き抜けた後、バンド内へ戻る（リ・エントリー） が出ると“パニック一巡”の形になりやすい。抜けたままだと“継続崩れ”。

MACD（minus-ok）：マイナス圏でも良いが、下げの加速が止まっているかが重要。MACDがさらに悪化し続けるなら“底打ち未確認”。

- **判断**：RSI≤20 ＋（下抜け→バンド内回帰）＋MACD悪化停止 → “底拾いの根拠が揃い始め”。どれか欠けるなら“危険域のまま”。

### ⑦ 【逆張り】VWAP回帰（平均回帰戦略）
適正価格（VWAP）から大きく乖離した株価が、磁石のように引き戻される性質を利用します。
```bash
tickwise -t <TICKER> -I --vwap --weight-vwap 3.0 --buy-rsi 35 --stance buyer
```
- **論理**: VWAPと現在値の乖離をスコアの最大根拠にします。行き過ぎたゴムが戻る瞬間を捉えます。

- **結果の見方**
VWAP乖離：価格がVWAPから大きく下に乖離しているほど“回帰余地”は増える。ただし乖離が拡大し続ける局面は“トレンド負け”の可能性。

VWAP回帰の確認：理想は「VWAPへ向かって乖離が縮小」→「VWAP近辺に到達」→「VWAPをまたぐ/上で維持」。この流れが出るほど成立。

RSI（35）：30ほど極端ではないが“弱り切ったところからの戻り”を狙う。35を回復できないなら回帰が鈍い可能性。

判断：VWAP乖離が“縮小方向”に転じ、RSIが35を回復し始める → “回帰の初動”。乖離が広がるままなら“まだ時期じゃない”。

### ⑧ 【防御】ホルダーの警戒モード
保有株の「逃げどき」を探る、ディフェンス重視の設定です。
```bash
tickwise -t <TICKER> -I --stance holder --sell-rsi 65 --macd-diff-mid 5 --bollinger --weight-bollinger 2.0
```
- **論理**: RSIの売り閾値を少し下げて早めに警告。バンド上限での過熱感を厳しめに評価させます。

- **結果の見方**
RSI（売り65）：65以上は“過熱寄り”。ホルダー視点では「上がったから強い」ではなく「上がり過ぎの警戒」トリガーとして使う。

MACD diff（--macd-diff-mid 5）：Tickwiseのdiffが 設定値を超える 方向なら“勢いが出過ぎ/振れ過ぎ”の可能性。過熱局面の警告として読む。

ボリンジャー：上側バンドにタッチ/張り付きは“過熱”の典型。そこからバンド内へ戻る動きが出ると“天井サイン”になりやすい。

- **判断（holder**）：RSI≥65 ＋ diffがmid超え ＋ 上側バンド反落 → “逃げどき候補が揃う”。逆にRSIが落ち着き、バンド内で安定なら“継続保有”寄り。

### ⑨ 【全方位】AIセカンドオピニオン
全56指標をフル稼働させ、AIに「指標間の不整合」を炙り出させます。
```bash
tickwise -t <TICKER> -I --ema --sma --bollinger --fibonacci --adx --ichimoku --vwap --llm
```
- **論理**: 人間では見きれない多角的データをAIにぶつけ、総合的な「穴」を指摘させるためのレーダーチャート設定。

- **結果の見方**
RSI（売り65）：65以上は“過熱寄り”。ホルダー視点では「上がったから強い」ではなく「上がり過ぎの警戒」トリガーとして使う。

MACD diff（--macd-diff-mid 5）：Tickwiseのdiffが 設定値を超える 方向なら“勢いが出過ぎ/振れ過ぎ”の可能性。過熱局面の警告として読む。

ボリンジャー：上側バンドにタッチ/張り付きは“過熱”の典型。そこからバンド内へ戻る動きが出ると“天井サイン”になりやすい。

- **判断（holder）**：RSI≥65 ＋ diffがmid超え ＋ 上側バンド反落 → “逃げどき候補が揃う”。逆にRSIが落ち着き、バンド内で安定なら“継続保有”寄り。

- **結果の見方**
最初にやること：各指標が“同じ方向を向いているか”だけを見る（順張り勢＝EMA/SMA/ADX/一目が強気、逆張り勢＝VWAP乖離やボリバン下限など）。

不整合の典型：トレンド系（EMA/SMA/一目/ADX）が強気なのに、ボリバンが過熱・RSI過熱・MACDが鈍化、など“短期と中期の矛盾”が出る。

フィボナッチ：価格がフィボ水準で止まりやすい（反発/反落しやすい）かを、他の支持抵抗（雲・SMA・VWAP・バンド）と“重なっているか”で見る（重なるほど強い壁）。

LLMの使い方：結論よりも「なぜ矛盾しているか」「どの指標が足を引っ張っているか」の説明を拾う。AIが言う“弱点”が、あなたの許容できるリスクかを判断材料にする。

判断：多数が同方向なら“自信度高め”。割れているなら“結論を急がず、どの時間軸の戦略で戦うか”を決め直す用途。

### ⑩ 【情報戦】材料・テーマ深掘り
テクニカルはベースに留め、ニュースの「中身」をAIに徹底的に読み込ませます。
```bash
tickwise -t <TICKER> -I --llm --news -q "業績予想 修正 増配" --weight-news 3.0 --news-count 20
```
- **論理**: カスタムクエリ `-q` で材料を絞り込み、AIの総評を「情報先行」の投資判断に活用します。

- **結果の見方**
ニュースの質チェック：同じテーマでも「一次情報（決算/IR）」「報道（解説）」「噂/観測」のどれかを分ける。一次情報ほど重く見る。

AI要約の見方：結論（買い/売り）より、要約内の“事実”と“解釈”を分けて読む。解釈が強いときは元ニュースの見出し/論点に戻る。

件数（news-count 20）：材料が“単発”なのか“複数ソースで裏が取れている”のかを見る。複数で同方向なら材料の確度は上がりやすい。

weight-news 3.0：ニュースがスコアに強く効く設計なので、テクニカルが弱いのにニュースだけで押し上がるケースが出る。その場合は“材料先行の短期戦”として割り切るか判断。

- **判断**：材料が明確で、複数ソースで補強され、AIがリスク（反証点）も挙げている → “情報戦として成立”。材料が曖昧/一発ネタなら“熱量はあっても不安定”。
