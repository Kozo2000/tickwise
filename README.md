📘 Tickwise — Stock Technical Analysis & News-Driven Insight Tool

Rust で書かれた、テクニカル指標＋ニュース要約＋スコアリングの統合 CLI ツール

「落ちるナイフをいつ掴む？」「今の反発は本物？」
そんな“個人投資家の永遠のテーマ”に、静かに寄り添う分析ツールです。

🚀 Tickwise とは

Tickwise は、株式の テクニカル分析 と ニュース要約 を
1 コマンドで統合して返す CLI ツール です。

MACD / RSI / SMA / EMA / Bollinger / Fib / ADX / ROC / VWAP / Ichimoku

Brave News API（自然言語検索/財務ワードフィルタ対応）

OpenAI（要約+解釈）

各指標のスコア合算 → Buyer / Seller / Holder の立場別に最適化

CSV/JSON ログや、ニュース件数/鮮度/抽出語彙など細かい調整可能

コマンドオプションは 56 種類

Rust 製で、依存性少なめ・低リスク構成。
個人向けツールですが、市販品質に寄せて設計しています。

✨ 特徴とアピールポイント（軽めに、誇張せず）
1. Rust のメモリ安全 × 個人ツールの安心感

安全性を最優先し、
unwrap / expect を原則禁止（すべて Result / Option で扱う）
という指針で実装しています。
投資判断の材料になるツールなので、動作の安定性は最重要と考えています。

2. 「設定が真のソース（SoT）」

Config と CLI 引数を中心に、
動作・重み付け・ニュース抽出・閾値・スコアリングすべてを外部化。

3. テクニカル指標の“立場別”スコアリング

一般的な分析ツールと異なり、
買い手 / 売り手 / 中立 の立場（stance）で
スコアの解釈を切り替えられます。

4. News × Technical のハイブリッド

Brave の自然言語ニュース検索
＋
独自の Tier 分類（価格影響：高/中/低/無）で
“今この銘柄に何が起きているか”が短時間で読めます。

5. 個人ツールとしては丁寧な品質管理

派手なことはしていませんが、

API 鍵は tickwise.env に完全分離

SBOM（Syft）生成

Release ビルドは SHA-256 付与

VirusTotal で誤検知をチェック

依存crateを絞り、攻撃面積を縮小

unwrap禁止 による安定動作

「市販ツールにも負けないくらいの丁寧さ」を1つの目標にしています。

🧭 なぜこの設計？（Philosophy）

Tickwise は、作者が Rust を学習しながら
“壊れにくく・扱いやすく・読みやすい CLI を個人でも作れる”
という挑戦のひとつとして生まれました。

全エラーは Result<T> で扱い、unwrap / expect 禁止

SoT（設定）主導で動作が変わる

「ニュース × テクニカル」の多視点モデル

再現性・透明性・安全性

“作者の感情より、指標とニュースの事実を淡々と提示する” を徹底
